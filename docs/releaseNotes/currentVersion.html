Release 3.04.0 Enhancements and Fixes

  • User Experience Improvements

    • Custom Policies: ZCSPM now allows you to customize default policies. You can clone or edit a default policy to meet your organization's requirements. For example, consider this security policy - Unused credentials are disabled after 90 days. You can clone and edit this policy to change the duration to meet your organizations risk profile. To learn more, see Customizing Security Policies
    • Override Risk Impact for a Security Policy: In addition to overriding a security policy status, ZCSPM now allows you to override security policy risk impact to tailor the risk associated with your organization's assets. For example, a security policy failure might be of high risk for your production environment, but of low risk for your test environment. You can override the policy for the test environment. To learn more, see Overriding Security Policies.
    • Asset Security Dashboard Enhancements:
      • You can now filter assets by tags, risk level, and other attributes and save it as a view. You can access this view to manage your filtered assets security and risk posture.
      • You can now assess the risk associated with a specific Resource Type. This risk is aggregated from all the resources present in the Resource Type.
      • You can now view the failed policies and the corresponding audit and remediation procedures from the Asset Details page.
    • Support for Google Kubernetes Engine (GKE): You can configure a ZCSPM agent for GKE to support the policies listed in the CIS GKE K8s 1.5.1 benchmark. To learn more, see Configuring ZCSPM Agent for Google Kubernetes Engine.

  • Platform & Stability Improvements

    • AWS data collection process now uses AWS config to support massive scale requirements for the AWS service using Auto Scaling groups.
    • Resolved the following issues:
      • Azure Resources in the Asset Dashboard now have tags.
      • Azure Disks Resource count is fixed.
      • Security policies for AWS::RDS::MYSQL::Instance can now collect resources.
      • The GCP Onboarding Health Status page now displays the correct prerequisite status for Cloud Asset Viewer Role.
      • GCP Organization Onboarding Health Status page now supports the Organization Viewer role.
      • The UI validations for data feed configurations are fixed.
      • The Health Status page now has updated help links for GCP Project-based onboarding.
      • The Private Benchmark CSV report displays the security policy data for updated configurations only after the subsequent scan.

    • Added support for the following benchmarks:
      • Center for Internet Security AWS Foundations Benchmark v1.3.0
      • Center for Internet Security GKE - Kubernetes Benchmark v1.5.1
